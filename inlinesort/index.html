A basic inline filter that uses the browser's CSS engine to select elements. It would be really nice if we could have made something this simple and functional using only CSS, but there are a couple of problems:

  1. the value of the input attribute alone is not enough: when it changes, we need the property (not available to CSS)
  2. We then need to take that value and use it as a selector elsewhere (not possible in CSS - and obviously, custom properties will not work here)

So, we need Javascript

And... we need one more thing: a data-search attribute on the element we wish to show/hide that contains the searchable text. To clarify, here is the (simplified) first element:

<div class="item" data-search="Proměnlivé křeslo">
  <div class="item-inner">
    <h3 class="item-heading">Proměnlivé křeslo</h3>
    ...
  </div>
</div>

This is pretty unobtrusive and easy to generate!

The Javascript

A couple of things to note:

  • The new style element is added on the DOM ready event here[1], so if you want to populate the search on load (eg. so that users can get a link to a specific search), then it's best to do this when rendering the HTML (server-side). If you must use JS, then trigger a keyup event after populating the element.
    1. It could be added earlier: when the element is ready (because the head element will be ready by this point), or even when the head element is available (but obviously the search.value will not be ready at this point)
  • The style modification which displays the results is made on the keyup[1] event, after the user has entered more than 2 characters.
    1. It might be nicer to use the change event, but this is only triggered when the input loses focus. Which is awful in this use case 😄

Why not simply show/hide the elements using JS rather than repeatedly writing a new (very small) stylesheet?

  • Hey, why not?
  • THIS NEEDS TESTING but I had the feeling that rather than writing to a number of DOM elements, amending a single text node in the head might be quicker. I also wondered whether using a CSS selector in JS might be slower than simply using CSS

The final result is something that should be pretty fast in most cases, and is very adaptable (eg add the data-search element to a <tr> and you get a nice table filter)
